I have always disliked using frameworks.

But, i was looking at htmx… maybe i even snuck a surreptitious glance or two at _hyperscript… and I started to get that hunger again, the one that makes you want to make things harder than they need to be.

I ended up reading through the htmx website when I stumbled upon a great jewel: htmz.

You see, the problem with htmx is that it is simply too large. 15.7 kB?!? That is like, the size of this HTML file. No, htmz is a much more respectable 166 bytes. That is smaller than this paragraph of text.

How does it do it? Well, htmz is a hidden <iframe/> set up so that — whenever it loads a page — it copies itself into a container in the parent document. So if you ever try to open a link with that iframe as the target, it instead overwrites some other part of the page with the content of that link.

It misuses anchor tags and form actions to load content, it entirely abuses URL hashes for container selection, it literally consists of a hidden second page that copies itself wholesale into the main page every time it is used — it is perfect.

Even once you think you have gotten a handle on it, it continues to be fucked in subtle ways that result in so many opportunities to implement fun and unique solutions.
